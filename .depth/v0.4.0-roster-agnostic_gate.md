USER_PATH:
  Scenario A (pip-install turnkey, no change): user runs `pip install amatelier`, `amatelier init` → defaults to curated-five template → `amatelier roundtable ...` → identical RT output to v0.3.0.
  Scenario B (customization): user runs `amatelier team new nova --model sonnet --role "..."` → cli.py adds nova to config.json workers + creates user_data_dir/agents/nova/CLAUDE.md from template → `amatelier roundtable` now runs with nova + existing workers → roster-agnostic engine spawns nova via _launch_claude (backend defaults to claude) → RT completes.
  Scenario C (alternative backend worker): user edits config.json to add `"research-bot": {"model": "gpt-4o", "backend": "openai-compat", "role": "..."}` → runner dispatch reads backend field → spawns research-bot via openai-compat path → RT completes.
  Scenario D (tests without API keys): contributor runs `AMATELIER_MODE=mock pytest tests/test_mock_backend.py` → MockBackend returns deterministic completions → integration tests validate engine paths (workers spawn, scoring dispatches, therapist cycles) without network.
