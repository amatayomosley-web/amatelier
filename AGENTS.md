# Amatelier — Agent Instructions

Generic instructions for any AI agent or coding assistant working in this repository. (Claude Code, Cursor, Copilot, Aider, Codex, or others.)

## What this project is

A self-evolving multi-model AI team skill for Claude Code

## Invariants

- **Amatayo Standard compliance.** CI enforces. Any PR that breaks structure will fail.
- **Hand-written sources:** `docs/` (narrative content), `src/`, `CLAUDE.md`, `llm/SPEC.md`, `llm/<domain>.md`.
- **Generated derivatives:** `llm/API.md`, `llm/SCHEMA.md`, `llms.txt`, `llms-full.txt`, `.cursor/rules/*.mdc`, `.github/copilot-instructions.md`. Don't hand-edit these — they are rebuilt by CI and changes will be overwritten.
- **`llm/` is flat** — no subdirectories.
- **Conventional commits** — `feat:`, `fix:`, `docs:`, `refactor:`, `test:`, `chore:`, with `BREAKING CHANGE:` footer for majors.
- **Tests and lint pass before PR** — `make test && make lint`.

## Two-layer paths (do not cross)

Amatelier ships as a pip package; the installed `src/amatelier/` tree must be treated as read-only at runtime.

- **Bundled layer** — `src/amatelier/` and everything inside the wheel. Canonical source; never written to from runtime code.
- **User data layer** — everything returned by the helpers in `amatelier.paths` (`user_data_dir`, `user_agent_dir`, `user_db_path`, `user_digest_dir`, `user_briefing_dir`, `user_store_ledger`, `user_novel_concepts`, `user_shared_skills_index`, `user_config_override`, `user_logs_dir`). All mutable state lives here.

When editing code:

- Route persistence, logs, ledgers, and agent memory through a `paths.user_*()` helper. Never hard-code writes under `src/amatelier/`.
- Per-agent seeds (`src/amatelier/agents/<name>/CLAUDE.md`, `IDENTITY.md`) are bundled. Seed edits only reach a user's environment after `amatelier refresh-seeds` or a fresh install.
- The generated-derivatives list above is regenerated by CI; hand edits will be discarded.

## Three LLM backend modes

All LLM access flows through `amatelier.llm_backend.get_backend()`. At runtime the abstraction resolves to one of:

- `claude-code` — `ClaudeCLIBackend`, used inside Claude Code with the `claude` binary on PATH
- `anthropic-sdk` — `AnthropicSDKBackend`, used when `ANTHROPIC_API_KEY` is set
- `openai-compat` — `OpenAICompatBackend`, used with `OPENAI_API_KEY`, `OPENROUTER_API_KEY`, or local Ollama

Override with `AMATELIER_MODE`. When adding LLM calls:

- Use the backend object returned by `get_backend()`. Do not shell out to `claude` or `import anthropic` at the call site.
- Any new capability must extend the `LLMBackend` Protocol in `src/amatelier/llm_backend.py` and be implemented by all three concrete backends.
- Register any new provider env var in `describe_environment()` so `amatelier config` surfaces it.

## Tests

- `tests/test_smoke.py` — pytest, import/CLI smoke checks, runs in CI
- `tests/test_refresh_seeds.py` — pytest, verifies seed materialization, runs in CI
- `tests/test_integration.py` — **standalone script**, exercises live LLM backends. Not pytest. Not run in CI. Run it manually when you change a backend.
